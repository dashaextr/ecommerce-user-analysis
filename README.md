# E-commerce User Behaviour & Retention Analysis

## Overview
This project analyses user behaviour and transaction data to understand conversion, retention, and revenue trends.

## Key Questions
- Which countries generate the most user registrations?
- How does conversion from registration to first order change over time?
- What is user retention (D1, D7, D30)?
- Are there users with abnormal transaction behaviour?
- How does average order value change week over week?

## Tools
- SQL (PostgreSQL)

## Key Insights
- Strong concentration of users in one region (~50%)
- Conversion remains around 50%, indicating onboarding improvement potential
- Retention drops significantly after Day 1
- Identified users with multiple failed transactions
- AOV shows high volatility due to low order volume

## Business Recommendations
- Improve onboarding flow to increase conversion
- Investigate retention drop after Day 1
- Analyse failed transactions and payment issues
- Increase order volume to stabilise revenue
